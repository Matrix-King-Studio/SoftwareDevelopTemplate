# Matrix工作室软件开发项目模板

## 分支维护

- master 分支：生产环境分支，用于存放最新的稳定版本，自动部署至生产服务器（横向/外包项目对应甲方服务器，纵向/自研项目对应内部 WEB 服务器），生产环境不允许个人开发分支合并，只允许 dev 分支合并，必须创建 [PR](https://github.com/Matrix-King-Studio/***/pulls) 提交合并请求，AI Code Review 和开发负责人 Code Review 之后没问题才能合并。
- test 分支：测试环境分支，用于存放最新的开发版本，自动部署至测试服务器（横向/外包项目对应工作室 WEB 服务器，纵向/自研项目对应个人服务器），测试环境允许个人开发分支合并。
- dev-[姓名首字母缩写] 分支：个人开发分支，用于存放个人的开发版本，例如 `dev-zs` 表示的就是张三的开发分支。
